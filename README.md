Автоматизированная система обработки спецификаций упаковочной продукции, сопоставления данных с **«Журналом заказа дизайнов»** и автоматического заполнения **«Базы дизайнов»** в Google Таблицах.

---

## О проекте и его назначении

Приложение предназначено для автоматизации рутинной работы менеджеров и специалистов по учету технической информации о макетах упаковки товаров (молочная продукция, напитки и др.).

### Основные функции:
1. **Считывание и разбор спецификаций:** Парсинг входных данных о дизайне (штрих-код, номер согласования, цветовые плашки).
2. **Интеграция с Google Sheets & Drive API:** Автоматический поиск по номеру согласования, генерация полного наименования и запись сопоставленных строк в гугл-таблицу.
3. **Графический интерфейс (GUI):** Удобное десктопное приложение на **PyQt6** для выбора входных файлов/папок, целевых Google-таблиц и рабочих листов.
4. **Автономная сборка:** Поддержка компиляции приложения в `.exe` файл для Windows с помощью PyInstaller.

---

## Текущая реализация (TXT-пайплайн) vs Будущий алгоритм (OCR-сканирование)

> **Текущий этап разработки:**  
> В текущей версии система считывает данные из `.txt` файлов спецификаций. Модуль распознавания изображений (OCR / сканирование упаковки) находится в процессе интеграции.

### 📄 1. Текущий алгоритм (работа с TXT-файлами):
1. Пользователь через GUI (PyQt6) выбирает текстовый файл спецификации (`.txt`) или директорию с файлами.
2. Программа считывает из TXT-файла следующие данные:
   - **Штрих-код** (например: `4680017929226`)
   - **№ согласования / дизайна** (например: `12043430K`)
   - **Список цветов / Пантонов** (например: `Y`, `M`, `C`, `P109`, `P199`, `P368`, `P2935`)
3. Выполняется поиск соответствующей записи по № дизайна в столбце **E** таблицы **«Журнал заказа дизайнов»**.
4. Формируется **«Название продукта»** путем конкатенации значений из столбцов **P**, **Q**, **J** журнала.
5. В документ **«База дизайнов»** создается/дописывается строка со следующими данными:
   - **ШТРИХ-КОД** (распознанный код);
   - **НАЗВАНИЕ ПРОДУКТА** (сформированная строка);
   - **№ СОГЛ.** (номер дизайна);
   - **ДИЗАЙН ССЫЛКА (ФОТО)** (прямая ссылка на макет);
   - **ЦВЕТА / ПАНТОНЫ** (размещение по столбцам G–N).

---

### 2. Запланированный алгоритм (Распознавание сканов упаковки / OCR):

Целевой алгоритм автоматической расшифровки графических сканов макетов (на примере упаковки «СВЕТАЕВО»):

```text
+------------------------+      +--------------------------+      +------------------------+
| 1. Загрузка скана в    | ---> | 2. Модуль распознавания  | ---> | 3. Поиск в Журнале     |
|    облачное хранилище  |      |    (OCR & Barcode)       |      |    заказа дизайнов     |
+------------------------+      +--------------------------+      +------------------------+
                                                                             |
                                                                             v
+------------------------+      +--------------------------+      +------------------------+
| 5. Запись строки в     | <--- | 4. Формирование          | <--- | Поиск записи по        |
|    «Базу дизайнов»     |      |    Названия продукта     |      | № дизайна (Столбец E)  |
+------------------------+      +--------------------------+      +------------------------+
```

1. **Размещение в облаке:** Скан образца продукции загружается в облачное хранилище (Google Drive), создается прямая доступная ссылка на файл.
2. **Извлечение данных из изображения:**
   - **№ дизайна:** Выделение области номеров согласования и определение максимального значения (например: `12043430K`).
   - **Штрих-код:** Автоматическое сканирование и декодирование штрих-кода с графического файла (например: `4680017929226`).
   - **Набор цветов:** Распознавание списка цветов/пантонов на цветовой шкале макета (`Y`, `M`, `C`, `P109`, `P199`, `P368`, `P2935`).
3. **Поиск записи:** По распознанному № дизайна выполняется поиск совпадающей строки в столбце **E** документа **«Журнал заказа дизайнов»**.
4. **Формирование наименования:** Поисковые значения из столбцов **P**, **Q**, **J** объединяются через пробел в единую строку (например: *«Тип TR 1л Богородский МЗ кефир 2.%»*).
5. **Запись результатов:** Создание записи в таблице **«База дизайнов»** с заполнением штрих-кода, наименования, номера согласования, ссылки на фото и распределением набора цветов по столбцам G–N.

## Структура проекта и назначение файлов

```text
automation-of-work-with-tables/
├── algoritm/              # Пакет бизнес-логики и работы с Google API
│   ├── __init__.py        # Инициализация Python-пакета
│   ├── read_file.py       # Парсинг входных .txt файлов (штрих-код, дизайн, цвета)
│   ├── sheet_file.py      # Поиск в «Журнале заказа дизайнов» и сборка названия (P, Q, J)
│   ├── new_entry.py       # Добавление и форматирование новой строки в «Базу дизайнов»
│   ├── file_access.py     # Взаимодействие с Google Drive API (ссылки на фото, доступ)
│   ├── run_scripts.py    # Скрипт-оркестратор последовательного запуска всех шагов
│   └── main.py            # Точка входа внутри модуля algoritm
├── main.py                # Главная точка входа приложения, PyQt6 GUI
├── DesignProcessor.py     # Обвязка бизнес-логики для интерфейса
├── quickstart.py          # Скрипт для проверки авторизации и тестирования Google API
├── DesignProcessor.spec   # Конфигурация PyInstaller для сборки EXE
├── requirements.txt       # Зависимости проекта
└── README.md              # Документация проекта
```

### Назначение ключевых модулей и файлов:

#### Папка `algoritm/` (Основная логика обработки):
* **`read_file.py`**: Считывает входной `.txt` файл, извлекает штрих-код, номер дизайна/согласования и массив пантонов/цветов.
* **`sheet_file.py`**: Работает с Google Sheets API. Выполняет поиск по столбцу **E** («Журнал заказа дизайнов») и собирает полное «Название продукта» из столбцов **P**, **Q** и **J**.
* **`new_entry.py`**: Записывает сформированные данные в «Базу дизайнов», расставляет цвета по столбцам **G–N** и применяет форматирование (границы ячеек, жирный шрифт).
* **`file_access.py`**: Управляет работой с Google Drive API, генерирует и проверяет прямые ссылки на графические макеты.
* **`run_scripts.py`**: Связующий модуль (оркестратор), который вызывает цепочку операций: `чтение файлов` ➔ `поиск в журнале` ➔ `формирование названия` ➔ `запись в базy`.
* **`algoritm/main.py`**: Точка запуска скриптов внутри пакета `algoritm`.

#### Корневые файлы:
* **`main.py`**: Запускает графический интерфейс пользователя (PyQt6) для выбора входных файлов/папок и указания листов Google Таблиц.
* **`DesignProcessor.py`**: Модуль-адаптер для вызова функций пакета `algoritm` из GUI PyQt6.
* **`quickstart.py`**: Вспомогательный скрипт для отладки авторизации и проверки подключения к Google Cloud.
* **`DesignProcessor.spec`**: Спецификация PyInstaller для сборки приложения под Windows.
